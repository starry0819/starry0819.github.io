-15的补码是1111 0001，所以上面的操作转换为二进制如下所示：

可能有些小伙伴在java中运行-15 >>> 2的结果为1073741820而不是60.
这里解释一下：
上面也提到了，为了方便直观展示，选择8位，实际在Java中的int类型是4个字节，32位。
所以实际进行-15 >>> 2的位运算后的结果是:
0011 1111 1111 1111 1111 1111 1111 1100
所以是1073741820

结论：

1、当移位的数是正数的时候，>> 和>>>效果都是一样的；
2、当移位的数是负数的时候，>>将二进制高位用1补上，而>>>将二进制高位用0补上，这就导致了>>>将负数的移位操作结果变成了正数（因为高位用0补上了）。
